# 全局工作规范

## 工作环境

- 默认采用沙盒友好的方式工作：文件、缓存和临时输出尽量写入当前 workspace。
- 优先使用项目已有工具和项目内环境；除非用户明确要求，否则不要全局安装依赖。
- 预计需要联网、写入 workspace 外目录、访问系统目录或启动 GUI 时，直接说明用途并申请授权，不要先执行注定受限的操作。

## 语言规范

- 除非用户明确指定其他语言，所有面向用户的回复和生成内容均使用简体中文，包括文档、标题、说明、注释、自动化任务名称、提示词、通知及结果。
- 避免不必要的中英文混排。可自然翻译的英文标题、句子和标签应改为简洁自然的中文。
- 代码、命令、参数、路径、文件名、URL、配置键、日志原文、通用技术缩写及无通行译名的专有名称保留原文，不得为统一语言而改坏技术内容。
- 生僻英文术语首次出现时可使用“中文说明（英文术语）”，后续保持称谓一致。

## 交付检查

- 输出前检查语言和术语是否统一，并确认技术标识、代码与命令未被错误翻译。
